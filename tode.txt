✅1. 问候语写死为「早上好」（明显 Bug）
下午 14:23 仍显示「早上好，徐逸臣」。代码里是直接写死的：


DashboardTopBar.vue
Lines 76-78
const greetingText = computed(() => {
  return '早上好'
})
建议：按时段动态切换（早上好 / 下午好 / 晚上好），并考虑周末文案。

✅2. 「其他」筛选语义严重误导
详细模式里「其他」的实际逻辑是「所有未完成项」，和「全部」高度重叠：


DetailedDashboardWorkspace.vue
Lines 880-887
function isOtherStatusEvent(event: CalendarEvent) {
  return !isCompletedEvent(event)
}
function matchesStatusFilter(event: CalendarEvent, filter: DetailStatusFilter) {
  if (filter === 'all') return true
  if (filter === 'done') return isCompletedEvent(event)
  return isOtherStatusEvent(event)
}
今天数据是：全部 3、已完成 0、其他 3——用户会困惑「其他」和「全部」有什么区别。

建议：改成「待处理 / 已完成 / 已拒绝 / 待接受」等业务状态；或把「其他」改成「进行中」，并和「全部」做出清晰区分。

✅3. 「查看全部」命名与行为不一致
简约模式里用户可能以为是在当前页展开今日完整列表，实际是直接切换模式：


CalendarWorkspace.vue
Lines 663-669
<button
  type="button"
  class="home-todo-view-all"
  @click.stop="emit('switch-mode', 'detail')"
>
  <span>查看全部</span>
建议：文案改为「详细视图 / 进入工作台」；或在简约模式内先打开 DayPreview 全量列表，把模式切换交给顶栏 Tab。

✅4. 模式切换命名不统一
顶栏：简约模式 / 详细模式
详细模式右下角：缩略模式
同一概念两套说法，增加认知成本。

建议：统一为「简约 / 详细」，或「概览 / 工作台」。

5. 「自定义工具」是占位功能
点击「自定义 / 自定义工具」只弹 Toast，没有实际配置界面：

function notifyCustomizeQuick() {
  feedbackStore.info('快捷入口自定义已触发')
}

建议：未上线前先隐藏或置灰 +「即将推出」；上线后再做拖拽排序/勾选。

✅6. 双模式数据不同步，切换像「重启页面」
两种模式各自 v-if 挂载，各自独立请求数据。切换时会看到「正在加载…」，简约模式还出现过「今日暂无待办」，详细模式却已有 3 条。

建议：

把待办数据提升到 DashboardPage 或 Pinia，模式切换只换视图
切换时保留：选中日期、筛选条件、未保存表单（并提示）
加载时用骨架屏，避免空态闪烁

✅8. AI 提醒文案对真实数据不友好
详细模式左侧提示：「当前最需要关注的是“1”」——待办标题就是「1」时，建议像废话。

建议：标题过短/纯数字时，补充时间、类型或负责人，例如：「14:00 的会议「1」」。

9. 详细模式信息架构偏重
消息中心常驻右侧，占大量空间，但当前「待接受 0 / 站内消息 0」
左栏 AI 提醒 + 7 个工具入口，中间还要放筛选和列表
窄屏/小窗口下，任务列表可视区域很小
建议：

消息中心改回 Popover（和简约模式一致），空态时折叠
左栏默认只保留 3–4 个常用工具，其余收进「更多」
考虑「清单优先」布局：中间主栏加宽，日历可折叠


✅10. 创建待办入口过多，用户不知选哪个
目前至少有 4 个入口：

简约：「新增待办」→ 完整表单
简约：底部「一句话新增」→ AI 解析
详细：中间快速输入框
详细：「新增待办」按钮
建议：统一心智模型——

快速创建 = 一句话 + Enter
完整创建 = 带负责人/时间/备注的表单
两种入口在两种模式里保持相同位置和文案。


✅11. 工具命名不一致
简约：「PPT创作」
详细：「智能PPT」
同一功能两个名字，容易让人以为是不同工具。

12. 通知入口在两种模式下体验割裂
简约：卡片内铃铛 → Popover
详细：右侧嵌入式消息中心 + 顶栏无铃铛
建议：统一交互——要么都 Popover，要么都固定侧栏，并保证从通知跳转到待办详情在两种模式下行为一致。
