# 待办实时同步优化方案（前端）

> 文档版本：v1.0  
> 创建日期：2026-07-22  
> 适用范围：PC 首页（简约/详细模式）、H5 移动端  
> 关联文档：[完整版](./待办实时同步优化方案-完整版.md) · [后台方案](./待办实时同步优化方案-后台.md)  
> 前置依赖：后台 P0 起提供 `todo_changed` WebSocket；P1 起提供 `changes` 接口

---

## 1. 目标

前端建立统一的**数据同步层**，消费后台 `todo_changed` 事件和 `changes` 补偿接口，实现：

- 日历/会议列表实时更新，无需手动刷新页面
- 打开中的详情面板跟随远端变更
- 消息中心与待办数据联动一致
- WebSocket 断线时通过 changes 接口最终一致
- PC 与 H5 共用同一套 sync 逻辑

## 4. 新增模块

### 4.1 `dashboard-sync.service.ts`

**路径**：`src/modules/home/dashboard/services/dashboard-sync.service.ts`

**职责**：

| 能力 | 说明 |
| --- | --- |
| WebSocket 连接 | 复用 `buildSysMessageWebSocketUrl`，进首页即 connect |
| 消息分发 | 解析 `sys_message` 和 `todo_changed` |
| 事件去重 | `eventId` Set，上限 1000 条 FIFO 淘汰 |
| 断线重连 | 指数退避，最大 30s |
| changes 补偿 | WS 断开时轮询 `GET /smart-todo/changes` |
| 订阅模式 | 允许 inbox / store / detail 注册 listener |

**核心类型**：

```typescript
export type TodoChangedEvent = {
  type: 'todo_changed'
  eventId: string
  action: 'created' | 'updated' | 'deleted' | 'status_changed' | 'assignee_added' | 'assignee_removed' | 'revoked' | 'process_added'
  entityType: 'todo' | 'meeting'
  todoId: string
  mainTodoId?: string
  childTodoId?: string
  affectedUserIds: string[]
  dateRange: { startDate: string; endDate: string }
  version?: number
  updatedAt: string
  canOpenDetail?: boolean
  snapshot?: Record<string, unknown>
}

export type SyncListener = {
  onTodoChanged?: (event: TodoChangedEvent) => void
  onSysMessage?: (message: SysMessage) => void
  onConnectionStateChange?: (state: 'connected' | 'disconnected' | 'reconnecting') => void
}
```

**对外 API**：

```typescript
export const dashboardSyncService = {
  connect(userId: string): void
  disconnect(): void
  subscribe(listener: SyncListener): () => void
  getLastSyncedAt(): string | null
  setLastSyncedAt(time: string): void
  syncChanges(since?: string): Promise<TodoChangesResult>
}
```

### 4.2 `useDashboardDataSync.ts`

**路径**：`src/modules/home/dashboard/composables/useDashboardDataSync.ts`

**挂载位置**：`DashboardPage.vue`（`onMounted` / `onBeforeUnmount`）

**选项**：

```typescript
type UseDashboardDataSyncOptions = {
  getLoadedRange: () => TodoDateRange | null
  getOpenDetailId: () => string | null
  onDetailInvalidate?: (payload: { todoId: string; reason: string }) => void
  onUnauthorized?: () => void
}
```

**核心逻辑伪代码**：

```typescript
function handleTodoChanged(event: TodoChangedEvent) {
  const loadedRange = options.getLoadedRange()
  if (!loadedRange || !isRangeOverlapping(event.dateRange, loadedRange)) return

  if (event.action === 'deleted' || event.action === 'revoked') {
    store.removeEvent(event.todoId)
  } else if (event.snapshot) {
    store.upsertFromSnapshot(event.todoId, event.snapshot)
  } else {
    void store.reloadEvent(event.todoId)
  }

  // 详情跟随
  if (options.getOpenDetailId() === event.todoId) {
    if (event.canOpenDetail === false) {
      options.onDetailInvalidate?.({ todoId: event.todoId, reason: 'revoked' })
    } else {
      void detailCache.loadTaskDetail({ id: event.todoId }, { force: true, silent: true })
    }
  }

  // 待接受列表（状态变更类）
  if (['created', 'status_changed', 'assignee_added'].includes(event.action)) {
    void notifications.refreshPendingTodos({ silent: true })
  }

  dashboardSyncService.setLastSyncedAt(event.updatedAt)
}
```

**补充行为**：

| 触发 | 动作 |
| --- | --- |
| `visibilitychange` → visible | 立即 `syncChanges(lastSyncedAt)` |
| WebSocket reconnect | 立即 `syncChanges(lastSyncedAt)` |
| WS disconnected > 10s | 启动 changes 轮询（30s，仅 visible） |

### 6.2 远端更新检测

```typescript
function isRemoteNewer(cached: CachedDetail, event: TodoChangedEvent): boolean {
  if (event.version != null && cached._cachedVersion != null) {
    return event.version > cached._cachedVersion
  }
  return event.updatedAt > (cached._cachedUpdatedAt ?? '')
}
```

## 9. H5 改造

### 9.1 新增 `h5/composables/useH5DataSync.ts`

复用 `dashboard-sync.service`，监听 `todo_changed`：

```typescript
export function useH5DataSync(options: {
  getWeekRange: () => TodoDateRange
  onTodoChanged: (event: TodoChangedEvent) => void
  currentUserId: () => string
}) {
  onMounted(() => {
    dashboardSyncService.connect(options.currentUserId())
    return dashboardSyncService.subscribe({
      onTodoChanged: (event) => {
        if (!isRangeOverlapping(event.dateRange, options.getWeekRange())) return
        options.onTodoChanged(event)
      },
    })
  })
}
```
