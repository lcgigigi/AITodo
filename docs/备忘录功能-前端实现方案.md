# 备忘录功能 — 前端实现方案（精简版）

> 文档版本：v2.0  
> 更新日期：2026-07-23  
> 原则：**UI 独立模块，数据层扩展 `todo.service.ts`，走 `/smart-todo/*` + `recordMode=2`**  
> 关联：[备忘录功能-后台接口方案](./备忘录功能-后台接口方案.md) · [frontend-code-index.md](./frontend-code-index.md)

---

## 1. 目标

- 独立「备忘录」页面与首页钉住区，**不上日历**
- 创建/改/归档/接受/拒绝 **复用待办 service**，传 `recordMode: 2`
- 仅 **2 个新 list 接口** 由 service 封装：`memo-list`、`archived-list`

---

## 2. 信息架构

```text
首页
└── MemoPinnedStrip          → loadMemoList({ scope: 'pinned' })

顶栏 → /memos
├── Tab 我的                 → scope: 'created'
├── Tab 收到的               → scope: 'received' | 'pending'
└── Tab 已归档               → archived-list?recordMode=2

日历 / 今日待办（不动）
└── month-list / today-list  后台已过滤 recordMode=1

待办详情
└── 「删除」改「归档」        → deleteTodo / archiveTodo
```

---

## 3. 目录结构

UI 仍建议独立模块，**不单独建 memo.service.ts**，逻辑放在 `todo.service.ts` + 薄 composable。

```text
src/modules/memo/
├── MemoPage.vue
├── components/
│   ├── MemoPinnedStrip.vue
│   ├── MemoCard.vue
```

**不需要**：`vite.config` 新代理、`/smart-memo` 路径。

## 5. `todo.service.ts` 扩展

### 5.1 复用现有方法（传 recordMode）

| 前端方法 | 接口 | 备忘录要点 |
| --- | --- | --- |
| `createTodo(draft)` | POST `/smart-todo/create` | payload 加 `recordMode: 2`；可不填时间 |
| `updateTodo(payload)` | PUT `/smart-todo` | 加 `recordMode: 2`、`pinned` |
| `deleteTodo(id)` | DELETE `/smart-todo/{id}` | 语义为归档 |
| `loadTodoDetail(id)` | GET `/smart-todo/{id}` | 详情抽屉 |
| `acceptTodos(ids)` | POST `/smart-todo/accept` | 接受后刷新 pinned |
| `rejectTodo(id, reason)` | POST `/smart-todo/reject` | 同待办 |

建议封装别名，避免业务层散落 magic number：

```ts
export function createMemo(draft: MemoDraft) {
  return createTodo({ ...buildMemoPayload(draft), recordMode: RECORD_MODE_MEMO })
}
export function updateMemo(...) { ... }
export function archiveMemo(id: string) {
  return deleteTodo(id)
}
```

### 5.2 新增 list 封装（仅 2 个 HTTP path）

```ts
export type MemoListScope = 'created' | 'received' | 'pending' | 'pinned' | 'archived'

export async function loadMemoList(params: {
  scope: MemoListScope
  page?: number
  pageSize?: number
}) {
  // GET /smart-todo/memo-list?scope=...
}

export async function loadArchivedItems(params: {
  recordMode: 1 | 2
  page?: number
  pageSize?: number
}) {
  // GET /smart-todo/archived-list?recordMode=...
}
```

| 调用方 | 方法 |
| --- | --- |
| 首页钉住条 | `loadMemoList({ scope: 'pinned' })` |
| 备忘录 · 我的 | `scope: 'created'` |
| 备忘录 · 收到的 | `scope: 'received'` |
| 备忘录 · 待接受 | `scope: 'pending'` |
| 备忘录 · 已归档 | `loadArchivedItems({ recordMode: 2 })` |
| 待办 · 已归档 | `loadArchivedItems({ recordMode: 1 })` |

### 5.3 适配层

`mapBackendItemToMemo()`：从 `SmartTodoBackendItem` 映射，校验 `recordMode === 2`。  
列表/详情 **禁止** 对 `recordMode=1` 走备忘录 UI。

## 6. 页面与组件要点

### 6.1 MemoPinnedStrip

- 数据：`loadMemoList({ scope: 'pinned' })`
- 最多展示 5 条；复制、取消钉（`updateMemo({ pinned: 0 })`）、点进详情
- 挂在 `CalendarWorkspace.vue` 今日待办 **上方**

### 6.2 MemoPage `/memos`

| Tab | 数据 |
| --- | --- |
| 我的 | `scope: 'created'` |
| 收到的 | `received` + 子筛选 `pending` |
| 已归档 | `loadArchivedItems({ recordMode: 2 })` |

新建：`MemoCreateDialog` → `createMemo()`  
**无**完成按钮、无进展、无日历字段（时间可隐藏）。

### 6.3 MemoDetailDrawer

按 `status` + 角色显示：编辑、归档、钉住、接受、拒绝、复制。  
**若 `recordMode !== 2` 误打开，立即关闭并提示。**

### 6.4 通知

- `notification.inbox.ts`：`bizType === 3` → kind `memo`
- WebSocket 收到 memo 消息 → `loadMemoList({ scope: 'pending' })` + `scope: 'pinned'`
- **不要** 调用 `loadTodos()` / 刷新日历

## 9. 测试要点

**单测** `todo.service`：create/update payload 含 `recordMode: 2`；`loadMemoList` / `loadArchivedItems` query 正确。

**场景**

1. 创建备忘录 → 不出现在 calendar month-list  
2. 派发 → 接受 → 首页 pinned  
3. 发送方编辑 → 接收方可见更新  
4. 归档 → `archived-list?recordMode=2` 有、`recordMode=1` 无  
5. 待办归档 → 仅 `recordMode=1` 列表
