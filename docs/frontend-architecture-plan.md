# 前端架构维护计划

更新时间：2026-07-09

## 当前结论

项目已具备模块化路由、严格 TypeScript、Pinia 状态层、请求拦截器和单元测试基础。当前主要维护
成本来自巨型工作台组件、两种工作台模式的重复编排，以及部分业务数据仍依赖静态配置。

## 已完成

- 删除无引用组件、图片、一次性脚本、构建包、截图、缓存和重复文档。
- 统一使用 pnpm 锁文件并移除未使用依赖。
- 修复 ESLint 门禁并统一 Prettier 格式。
- 待办范围加载只允许最新请求提交结果，避免快速切换月份或周时旧响应覆盖新数据。
- 请求错误保留 HTTP 状态、业务码、traceId 和原始异常。
- 过滤缺少 ID、有效日期或标题的异常待办数据，不再生成随机可操作 ID。
- localStorage 不可用时安全降级。
- 合并 Token 格式化逻辑和待办变更错误处理。
- ECharts 改为按需注册，显著缩小共享构建块。
- 2026 特殊日期已从页面组件移至独立数据文件。

## 后续维护原则

1. 工作台重构优先抽取纯计算、请求编排和 composable，模板 DOM 与 scoped CSS 分批保持不变。
2. 新增后台字段时先扩展适配层和测试，不让原始响应结构直接进入页面组件。
3. 新增年份前补充特殊日期数据；长期方案是由后台提供日期范围接口。
4. 不在 `VITE_*` 中保存生产密钥；当前登录预填兼容行为后续应单独迁移。
5. 每次变更必须通过 typecheck、ESLint、Vitest 和 production build。

## 目标

- 核心页面脚本逐步控制在可审查范围内。
- 共享逻辑保持单一实现，不在简约/详细模式中复制请求规则。
- 异常数据不进入可操作 UI。
- 仓库只跟踪源码、必要文档和真实业务资产。
